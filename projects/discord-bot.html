Self-Hosted Discord Bot on AWS

A personal software project outside of coursework, built to get hands-on with cloud infrastructure and full-stack ownership of a live application.

My role: solo developer, designed, built, and maintain the entire system

What It Does

The bot runs in a Discord server and provides a set of games and fun, interactive commands for members to use, the kind of lightweight engagement features that keep a community active. It's a persistent, always-on service rather than a script that runs once, which is what made the hosting and reliability side of the project as interesting as the bot logic itself.

Why I Built It

As a materials engineering student, most of my technical work is lab-based or simulation-based. This project was a deliberate way to build comfort on the software and infrastructure side, writing an application in Python, deploying it to a real cloud environment, and being responsible for keeping it running, rather than working from a pre-configured lab machine or a one-off script.

How It's Built

The bot itself is written in Python using Discord's bot API to listen for and respond to commands in real time. It's deployed on an AWS instance that I provisioned and configured myself, handling everything from initial setup through ongoing uptime monitoring and maintenance. Running it as a continuously live service, rather than a local script, meant thinking about reliability, restarts, and monitoring in a way that project-based coursework rarely requires.

Python Discord API AWS EC2 Linux

Outcome

The bot has been running independently since deployment, giving me practical, ongoing experience with cloud systems administration alongside my materials engineering coursework, a small but real example of being comfortable working across both hardware and software domains.
